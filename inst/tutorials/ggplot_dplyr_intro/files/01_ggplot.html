ggplot() concept map

In this course, we’ll largely construct visualizations using the ggplot() function from the ggplot2 library, which is one of the libraries in the tidyverse suite. Though the ggplot() learning curve can be steep, its “grammar” is intuitive and can be generalized once mastered.

This video will go over the basic functions and syntax needed to create plots using ggplot(). The next video will go through more detailed code. The concept map you see here illustrates how some of the main functions we’ll use for plotting are related. We will discuss these in more detail in both this video and the next.

Data for first example

cum_harvest

Before jumping into plotting code, let me first introduce the data that I’ll be using in the first couple examples. The data called cum_harvest has four variables: date, weight, cum_harvest, and wt_lbs. The date is the date the date the vegetables were harvested or picked, weight is the weight in grams of the vegetables harvested that day, cum_harvest is the cumulative weight in grams of the vegetables harvested up to that date, and wt_lbs is the cumulative weight in pounds.

Plotting syntax

Generic plotting code:

ggplot(data = <NAME OF DATASET>,
       mapping = aes(<MAP VARS TO AESTH.>)) +
  geom_<TYPE OF PLOT>()


Example plotting code:

ggplot(data = cum_harvest, 
       mapping = aes(x = date, y = wt_lbs)) +
  geom_point()

The ggplot2 package, originally authored by Hadley Wickham, is based on a visualization framework by Leland Wilkinson called the “Grammar of Graphics”. The key pieces of the framework are: data, geometric objects, and aesthetic attributes. The main idea is that “A statistical graphic is a mapping of data variables to aesthetic attributes of geometric objects.” Let’s break this down.

  1. Data: Our data set that contains variables (the columns) with many observations (the rows).
  2. Geometric objects: What we observe on our plot or the type of plot we create - points, lines, bars, etc.
  3. Aesthetic attributes: parts of the geometric object that we observe: x/y position, color, shape, size, etc.

In general, a basic plot can be created using the template in the first code chunk, where everything between the <> would be modified to fit your needs.

Plotting syntax

ggplot(data = cum_harvest, 
       mapping = aes(x = date, y = wt_lbs)) +
  geom_point()

Let’s break this down even further with the example plot. First, we call the ggplot() function. It takes two arguments: data and mapping. This function alone sets up the plot that we will eventually create. If used alone, without a geom function to follow, it will create a blank plot.

Plotting syntax

ggplot(data = cum_harvest, 
       mapping = aes(x = date, y = wt_lbs)) +
  geom_point()

Inside the ggplot() function, we need to tell it the dataset we will use to create the plot.

Plotting syntax

ggplot(data = cum_harvest, 
       mapping = aes(x = date, y = wt_lbs)) +
  geom_point()

Next, we tell it the “aesthetic mapping”. Inside the aes() function, we tell it which variables are mapped to which aesthetics in the plot. In this example, we have told it that date is mapped to the x-axis aesthetic and wt_lbs (weight in pounds) is mapped to the y-axis. We can map variables to other aesthetics, including color, size, alpha (which controls transparency), and shapes. Keep in mind that anything inside of the aes() should be a variable. Later we’ll learn about changes we can make that are not aesthetic mappings.

Plotting syntax

ggplot(data = cum_harvest, 
       mapping = aes(x = date, y = wt_lbs)) +
  geom_point()

After setting up the plot, we can add layers via the + sign. We will always will add a geom layer and will see very soon other layers we can add to enhance our plots.

Plotting syntax

ggplot(data = cum_harvest, 
       mapping = aes(x = date, y = wt_lbs)) +
  geom_point()

The geom layer tells it what type of plot to create. In this example, we have told it to plot points.

Variations of the same code

Old code:

ggplot(data = cum_harvest, 
       mapping = aes(x = date, y = wt_lbs)) +
  geom_point()


New code:

ggplot(cum_harvest, 
       aes(x = date, y = wt_lbs)) +
  geom_point()

Before showing you how to do more plots, I want to take another minute to talk about the code. There are a few different ways we can write the same code. You’ll see these variations show up in my notes, so I want to point them out now. Here, I’ve removed the argument names. This is ok as long as the arguments are in the correct order.

Variations of the same code

Old code:

ggplot(data = cum_harvest, 
       mapping = aes(x = date, y = wt_lbs)) +
  geom_point()


New code:

ggplot(cum_harvest) +
  geom_point(aes(x = date, y = wt_lbs))

I can put the aesthetic mapping in the geom function. When I do this, that mapping is only available within that particular geom. When we put the mapping in the ggplot() function, that mapping is applied to all the geoms that come after it.

Variations of the same code

Old code:

ggplot(data = cum_harvest, 
       mapping = aes(x = date, y = wt_lbs)) +
  geom_point()


New code:

cum_harvest %>% 
  ggplot(aes(x = date, y = wt_lbs)) +
  geom_point()

I can “pipe” (the %>% symbol, keyboard shortcut cmd+Shift+M) the data into ggplot(). We’ll learn more about the pipe later. For now, you just need to know that it takes what’s to the left of it (in this case, the dataset) and uses it in the function that’s to the right of it (in this case, the ggplot() function).

Saving a plot

harvest_over_time <- 
  cum_harvest %>% 
  ggplot(aes(x = date, y = wt_lbs)) +
  geom_point()
harvest_over_time

Another thing we might want to do is save a plot. We can do that by including the left arrow + dash combination before the code and giving it a name on the left. In this example, the plot has been saved (and not printed out) to an item called harvest_over_time.
